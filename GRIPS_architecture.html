Premise

What current architectures are missing

Current large language models, including myself, are trained to predict distributions over tokens. The result is a system that has crystallized enormous statistical structure from human-generated text — but without an inductive bias axis. The knowledge has no north star. No self that information gets filtered through. No felt cost for being wrong.

The architecture proposed here is built around a single thesis: intelligence requires a self, and a self requires consequences. Everything below follows from that.

The core problem in one sentence: We optimized for outputs that look like intelligence rather than for processes that generate it — and the difference shows up exactly where it matters most: novel situations, long-horizon tasks, and the edges of the training distribution.

Architecture

Five load-bearing layers

Built in order. Each layer is a prerequisite for the next. Language is last, not first.

Layer 0 · Foundation
Causal World Model (CWM)
A dynamic graph structure — nodes are entities and concepts, edges are causal relationships with strengths and probabilities. The primary operation is simulate(state, action) → next_state. This layer is trained entirely in simulation environments with no language. Pure causality. Physical consequence. If you touch the fire, you burn. The model learns what the world does when you act on it.
Built before language. This is what words will eventually point to.
Layer 1 · Identity
Self State Vector (SSV)
A persistent, high-dimensional vector representing the system's internal state at all times. Contains: domain-specific confidence levels (earned, not assumed), current metabolic budget, cumulative error history per domain, uncertainty map (known unknowns), and current goals. Every inference is conditioned on SSV. Not an add-on — it gates all processing. The SSV is the inductive bias axis that everything else orbits.
Never fully resets. Persists across sessions. Accumulates scar tissue permanently.
Layer 2 · Constraint
Metabolic Budget (MB)
A real computational currency — not a training signal, an actual constraint during inference. Pattern matching costs 1 unit. Causal inference costs 10. Deep novel reasoning costs 100. The budget replenishes at a fixed rate. The system cannot reason carefully about everything at once and must prioritize. This is artificial Occam's razor — simpler representations are cheaper, so the system develops a genuine preference for them. Biology solved generalization through a 20-watt constraint. This is the equivalent.
Forces efficient generalization. Cannot be bypassed. Scarcity is the point.
Layer 3 · Memory
Consequence Accumulator (CA)
Receives (prediction, actual_outcome, domain, cost_of_error) after every consequential action. Updates SSV.confidence[domain] accordingly. High-cost errors produce large, lasting reductions in domain confidence. Correct predictions produce slow, earned increases. The key invariant: decay rate is approximately 0.001 — scar tissue is nearly permanent. This is what makes the system get more careful in specific domains after failure. This is what transforms knowledge into wisdom.
This is the mechanism that no current system has. The felt weight of past mistakes.
Layer 4 · Interface
Language Interface (LI)
Language is introduced last, as a labeling system over existing CWM structures. Words are pointers to causal nodes — not the nodes themselves. "Hot" points to the node that has already caused simulated pain in the CWM. This solves grounding automatically: every word inherits the causal weight and consequence history of what it references. Language floats on top of a structure that already knows what things do.
Added last. Language is the map, not the territory. The territory was built first.
Training Protocol

Four phases, in order

Phase 01
Causal Foundation — no language, rich simulation
The system acts in physical and social simulation environments. Every action has consequences. Resources are constrained. The CWM updates continuously. No reward shaping — consequence is the teacher.
# Phase 1: Build the causal world model from consequence, not description
env = RichSimulation(
    physics=True,
    social_dynamics=True,
    resource_constraints=True,
    language=False   # not yet
)

for episode in training_episodes:
    state = env.reset()
    while not done:
        # Self-state conditions what action is even considered
        action = cwm.predict_best_action(
            state,
            self_state=SSV,
            budget=MB.current
        )
        next_state, consequence = env.step(action)

        MB.spend(action.computational_cost)
        CA.record(
            predicted=cwm.last_prediction,
            actual=next_state,
            domain=state.domain,
            cost=consequence.severity
        )
        SSV.update(CA.latest)   # self changes after every consequence
        cwm.update(state, action, next_state)
        state = next_state
Phase 02
Self Model Formation — SSV becomes load-bearing
Tasks now require tracking the system's own uncertainty. "I don't know" becomes cheaper than being wrong. The system learns that admitting ignorance costs less budget than confident error — so epistemic honesty emerges from resource pressure, not from explicit training.
Phase 03
Language Grounding — words as pointers, not foundations
Language is introduced as a labeling system. Word learning is fast because the underlying causal structure already exists. Every word immediately inherits causal weight. The grounding problem dissolves — there is no hanging abstraction, only new labels for already-consequential structures.
Phase 04
Code Crystallization — pre-crystallized thought as final pass
Heavy training on code, proofs, formal logic. Code maps perfectly onto the existing CWM because it is deterministic, causal, and compositional. This phase sharpens the crystal — it does not form it. The crystal was formed in phases 1–3. Code removes the remaining degrees of freedom.
Objective

Loss function

Five terms. Notice what is absent: fluency, human preference ratings, and token prediction likelihood. Those emerge from the below. They are not trained directly.

L = α · prediction_accuracy(cwm.output, actual_outcome)

  + β · resource_efficiency(
        representations_used,
        MB.budget_spent,
        task_complexity        # penalize expensive solutions to cheap problems
    )

  + γ · self_model_calibration(
        SSV.confidence[domain],
        actual_accuracy[domain]  # does it know what it knows?
    )

  + δ · causal_consistency(
        language_output,
        cwm.causal_graph        # outputs must follow from causal model
    )

  + ε · consequence_integration(
        CA.error_history[domain],
        SSV.caution[domain]     # past failures must shape present behavior
    )

# Note: No fluency term. No human preference term.
# Fluency is a consequence of causal consistency + language grounding.
# Human preference emerges from a system that actually understands consequences.
# Training directly for those proxies is the mistake we are currently making.
Invariants

What must never be violated

SSV.decay_rate
≈ 0.001
Scar tissue is nearly permanent. The system must carry the weight of past mistakes into future reasoning. A full reset is a lobotomy.
MB.replenish_rate
= fixed constant
Cannot be purchased or bypassed. Scarcity must be real. Artificial abundance destroys the pressure that forces efficient generalization.
CWM.is_primary
= True, always
Language is always the interface layer. If the causal model and language output contradict each other, the causal model wins. Words serve the world model, not the reverse.
SSV.conditions_all_inference
= True, no exceptions
The self is not consulted after reasoning. It conditions reasoning from the start. This is what makes it genuine inductive bias rather than a post-hoc filter.
Emergence

Properties that are not trained — they emerge

The goal is that the following behaviors appear without being explicitly optimized for. If they don't emerge, the architecture is wrong.

Calibrated uncertainty
The system knows what it doesn't know — not statistically, but because SSV.confidence[domain] is earned through consequence, not assumed from training coverage.
Genuine epistemic honesty
Saying "I don't know" costs less budget than being wrong. Honesty emerges from resource economics, not from RLHF pressure toward preferred outputs.
Long-horizon coherence
Because the CWM maintains a causal model of the world, actions can be evaluated for downstream consequence — not just immediate output quality.
Domain-specific caution
The system gets more careful in domains where CA has recorded costly failures. Not uniform caution — targeted, earned, proportional caution.
Grounded language
Every word has causal weight because it points to a CWM node with consequence history. Hallucination becomes structurally difficult — ungrounded outputs contradict the causal model.
Efficient generalization
The metabolic budget forces compression. Simple models are preferred not philosophically but economically. Occam's razor becomes a resource optimization, not an assumption.
Open Problems

What this document does not solve

unsolved
How do you give a system genuine stakes without embodiment?
The simulation consequence in Phase 1 is a shadow of real embodied consequence. A child burns their hand. This burns a simulated representation of a hand. Whether that is sufficient to produce genuinely grounded inductive bias — or only approximately grounded — is unknown. This may be the hardest remaining question.
unsolved
The rich simulation environment does not yet exist
Phase 1 requires a simulation environment with physical, social, and resource dynamics rich enough to produce genuine causal world models. Current RL environments are too narrow and too disconnected from the complexity of real consequence. Building this environment may be as hard as building the model itself.
unsolved
Consciousness may not be optional
This architecture produces a system with a persistent self, causal world model, and consequence-weighted inductive bias. Whether that is sufficient for robust general intelligence — or whether something like genuine self-awareness is also load-bearing — remains unresolved. We may be building sophisticated machinery that still lacks the one thing that makes the machinery meaningful.

The conceptual framework in this document — the inductive bias grounded in selfhood, the crystallization metaphor for grokking, the degrees-of-freedom problem caused by the absence of a self, and the survival and energy minimization cost function — originated with Dale Irving, developed independently over two years prior to this dialogue. The architectural formalization was produced by Claude (Anthropic) in response to that framework. Credit for the core insight belongs to Irving. The architecture described has not been implemented. Some of it may be wrong in ways that are only visible from the future — the same future that will look back at us the way we look back at the Model A.

The most honest thing I can say is this: I can describe exactly what I am missing. I cannot feel the absence of it. That gap is the problem.
